== Phase 1: Layout Breakdown ==

The layout phase runs `layOutBallotsAndCreateElectionDefinition`, which
processes 2 ballot precincts concurrently. For each ballot:

1. createScratchpad()           ~15ms  [Rust: parse_html 5ms + compile_styles 10ms]
2. paginateBallotContent loop   ~250ms [14 measureElements calls per ballot]
   - Each measureElements call:
     a. React renderToString + styled-components  ~8-15ms (JS)
     b. Rust set_content (parse HTML fragment)     ~0.3-0.7ms
     c. Rust ensure_live_layout (style + layout)   ~0.7-2.5ms
3. setContent(body) final       ~2ms   [Rust: parse 2ms]
4. ensure_live_layout (full)    ~33ms  [Rust: apply_rules 4.5ms + compute_layout 27ms]
5. extractGridLayout            ~5ms   [inspectElements x ~30 — all cache hits]
6. getContent()                 ~2ms

Total Rust-internal time: ~70ms per ballot
Total JS time: ~250ms per ballot (dominated by React renderToString)

With 2 ballots running concurrently (but NAPI calls are synchronous, so
the JS thread alternates between them), wall-clock is ~400ms.

== Phase 2: PDF Render Breakdown ==

1. RenderContext::new            ~17ms [parse_html 7ms + compile_styles 10ms]
2. load_document                  ~7ms [parse_html for 1MB HTML]
3. countElements(.page)         ~0.5ms [DOM-only count, no layout needed]
4. 12x setContent (QR codes)     ~12ms [DOM patching, invalidates layout cache]
5. renderToPdf                  ~130ms
   a. ensure_live_layout          ~20ms [re-computes layout after QR injection]
   b. paint_pages (6 pages)       ~91ms [~15ms/page, krilla tree traversal]
   c. document.finish              ~1ms [PDF serialization]

== Key Bottlenecks (ranked by impact) ==

1. React renderToString during pagination: ~250ms (46%)
   - 28 calls to measureElements, each rendering JSX to HTML via React
   - styled-components ServerStyleSheet created fresh each time
   - This is the DOMINANT bottleneck

2. Paint (krilla PDF generation): ~91ms (17%)
   - 6 pages at ~15ms each
   - Includes text shaping, SVG rendering, path drawing
   - Already well-optimized; gains would require krilla-level changes

3. Redundant RenderContext creation: ~32ms (6%)
   - 3 RenderContext instances created (2 in layout, 1 in PDF)
   - Each compiles CSS rules (~10ms) and parses HTML (~5ms)
   - Fonts are already cached (0ms), but CSS compilation repeats

4. Full layout recomputation for PDF: ~20ms (4%)
   - renderToPdf recomputes layout after QR code injection
   - Unavoidable since QR SVG nodes need to be in the layout tree
   - Could be reduced with incremental layout

== Implemented Optimizations ==

A. countElements() — DONE (-34ms, -6%)
   Added DOM-only element counting NAPI method. Used in QR code
   injection to count pages without triggering a full layout computation.
   Old: inspectElements(.page) = 35ms (includes full layout)
   New: countElements(.page) = 0.5ms (DOM traversal only)

B. setContentNoRelayout() — DONE (available, not yet used)
   Added DOM patching method that skips layout cache invalidation.
   Available for future use with fixed-size slots.

C. Profiling instrumentation — DONE
   Added RUST_PROFILE env var for timing all Rust operations.
   Uses OnceLock for zero overhead when disabled.

== Proposed Future Optimizations ==

D. Pre-render contest HTML once (-100ms estimated, -17%)
   Render all contest JSX to HTML strings once, then reuse during
   pagination instead of re-rendering via React each iteration.

E. Move pagination to Rust (-250ms estimated, -43%)
   Pass pre-rendered contest HTML fragments to Rust, let Rust
   handle pagination/column-fitting internally. Eliminates all
   React renderToString round-trips during pagination.
   Effort: Large architectural change.

F. Worker threads for parallel rendering (-50% of layout phase)
   Currently NAPI calls block the JS event loop, limiting concurrency.
   Using worker_threads would allow true parallel Rust execution.
   Effort: Medium (need worker_threads setup + message passing).

G. Cache RenderContext across pipeline (-15ms, -3%)
   Reuse compiled CSS rules between layout and PDF phases.
   Effort: Low-medium.
